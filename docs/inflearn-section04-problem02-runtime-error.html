Runtime Error 원인과 해결

Inflearn Section04 Problem02: 아나그램 비교에서 HashMap.get() 결과가 null일 수 있다.

현재 코드의 문제

  • s1의 문자 개수를 map에 저장한다.
  • s2를 돌면서 바로 map.get(x) - 1을 실행한다.
  • 그런데 s2에 s1에 없는 문자가 나오면 map.get(x)는 null이다.
  • null - 1을 계산하려 해서 NullPointerException이 발생한다.

해결 핵심

  • 빼기 전에 먼저 map.containsKey(x)를 검사한다.
  • 없는 문자라면 바로 NO를 반환한다.
  • 있는 문자라면 개수를 1 줄인다.
  • 개수가 0이 되면 map.remove(x)로 지워도 된다.

에러 재현 흐름

1. 입력 s1 = "abc"
s2 = "abd"
2. map 생성 {a=1, b=1, c=1}
d는 없다.
3. s2 순회 a, b는 처리된다.
다음 문자는 d다.
4. 런타임 에러 map.get('d')는 null.
null - 1은 불가능하다.
문제 지점: map.put(x, map.get(x) - 1);

현재 코드에서 위험한 부분

for (char x : s2.toCharArray()) {
    map.put(x, map.get(x) - 1);
    if (map.get(x) == 0) map.remove(x);
}

안전한 검사 순서

for (char x : s2.toCharArray()) {
    if (!map.containsKey(x)) {
        return "NO";
    }

    map.put(x, map.get(x) - 1);

    if (map.get(x) == 0) {
        map.remove(x);
    }
}
먼저 있는지 확인하고, 그 다음 개수를 줄인다.

최종 풀이 구조

public static String solution(String s1, String s2) {
    HashMap<Character, Integer> map = new HashMap<>();

    if (s1.length() != s2.length()) return "NO";

    for (char x : s1.toCharArray()) {
        map.put(x, map.getOrDefault(x, 0) + 1);
    }

    for (char x : s2.toCharArray()) {
        if (!map.containsKey(x)) return "NO";

        map.put(x, map.get(x) - 1);
        if (map.get(x) == 0) map.remove(x);
    }

    return map.isEmpty() ? "YES" : "NO";
}
