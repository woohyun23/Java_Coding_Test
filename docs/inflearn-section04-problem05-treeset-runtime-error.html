K번째 큰 수 풀이 정리

Inflearn Section04 Problem05: 배열 인덱스로 직접 접근하면 런타임 에러가 날 수 있고, 중복 합을 제거해야 한다.

기존 코드의 문제

  • 모든 3장 합을 int[] tot에 저장한다.
  • Arrays.sort(tot) 후 tot[t - m]으로 K번째 큰 값을 꺼낸다.
  • m이 가능한 합 개수보다 크면 t - m이 음수가 된다.
  • 같은 합이 여러 번 나오면 중복된 값도 각각 다른 순위로 센다.

해결 핵심

  • TreeSet에 3장 합을 넣는다.
  • Set이므로 같은 합은 자동으로 한 번만 저장된다.
  • Collections.reverseOrder()로 큰 값부터 정렬한다.
  • 앞에서부터 세다가 m번째 값을 만나면 반환한다.

런타임 에러 흐름

1. 입력 n = 3, m = 50
카드는 1 2 3
2. 조합 개수 3장을 뽑는 방법은 1개뿐이다.
t = 1
3. 인덱스 계산 t - m
1 - 50 = -49
4. 배열 접근 tot[-49] 접근 시도.
음수 인덱스라서 런타임 에러.
문제 지점: int res = tot[t - m];

중복 때문에 틀리는 예시

입력:
5 3
10 10 10 9 8

조합별 합을 그대로 나열하면:
30, 29, 29, 29, 28, 28, 28, 27, ...

중복 제거 후 순위:
1등: 30
2등: 29
3등: 28
4등: 27

정답은 28이다.
이 문제의 순위는 같은 합을 여러 번 세지 않는 기준으로 봐야 한다.

TreeSet이란?

  • TreeSet은 Set 계열 자료구조라서 중복 값을 저장하지 않는다.
  • 값을 넣을 때 자동으로 정렬된 상태를 유지한다.
  • 기본 정렬은 오름차순이다. 예를 들어 1, 5, 3을 넣으면 순회 시 1, 3, 5가 나온다.
  • new TreeSet<>(Collections.reverseOrder())를 쓰면 내림차순이 된다.

배열과 TreeSet 비교

방식 특징 이 문제에서의 문제점 또는 장점
int[] + Arrays.sort 모든 값을 그대로 저장한 뒤 정렬한다. 중복 합이 제거되지 않는다. tot[t - m] 접근 시 인덱스 에러가 날 수 있다.
TreeSet 중복을 제거하고 정렬 상태를 유지한다. K번째로 큰 서로 다른 합을 앞에서부터 안전하게 셀 수 있다.

TreeSet 사용 흐름

1. Set 생성 TreeSet<Integer> set을 내림차순으로 만든다.
2. 세 장 선택 i, j, k 반복문으로 모든 조합을 만든다.
3. 합 저장 set.add(sum)을 호출한다. 중복 합은 자동으로 무시된다.
4. K번째 찾기 큰 값부터 순회하면서 m번째 값을 반환한다.

최종 코드 구조

public static int solution(int n, int m, int[] arr) {
    TreeSet<Integer> set = new TreeSet<>(Collections.reverseOrder());

    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            for (int k = j + 1; k < n; k++) {
                set.add(arr[i] + arr[j] + arr[k]);
            }
        }
    }

    int cnt = 0;
    for (int x : set) {
        cnt++;
        if (cnt == m) return x;
    }

    return -1;
}

기억할 포인트

  • HashSet은 중복 제거는 되지만 정렬은 보장하지 않는다.
  • TreeSet은 중복 제거와 정렬을 동시에 처리한다.
  • K번째 값을 찾아야 할 때는 배열 인덱스로 무리하게 접근하기보다, 실제로 존재하는 값만 순회하면서 세는 방식이 안전하다.
