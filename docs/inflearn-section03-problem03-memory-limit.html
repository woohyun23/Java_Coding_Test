Memory Limit Exceeded 원인과 해결

Inflearn Section03 Problem03: 최대 매출 문제는 부분 배열을 저장하지 말고, 합만 이동시켜야 한다.

현재 코드의 문제

  • 반복문마다 new int[m]으로 임시 배열을 새로 만든다.
  • 각 시작 위치마다 m개를 다시 복사해서 O(n * m)이 된다.
  • 합도 temp[0] + temp[1] + temp[2]처럼 3개로 고정되어 있어 m이 바뀌면 틀린다.

해결 핵심

  • 임시 배열 temp는 필요 없다.
  • 처음 m개의 합을 구한다.
  • 창을 한 칸 옮길 때 새 값은 더하고, 빠지는 값은 뺀다.
  • 매번 max만 갱신한다.

슬라이딩 윈도우 흐름

1. 첫 구간 arr[0]부터 arr[m - 1]까지 더한다.
2. max 초기화 첫 구간의 합을 현재 최대값으로 둔다.
3. 한 칸 이동 오른쪽 새 값은 더하고 왼쪽 이전 값은 뺀다.
4. 반복 끝까지 이동하면서 max를 갱신한다.
다음 합 = 이전 합 - 빠지는 값 + 새로 들어오는 값

예시

m = 3
배열: 12 15 11 20 25

첫 합: 12 + 15 + 11 = 38
다음 합: 38 - 12 + 20 = 46
다음 합: 46 - 15 + 25 = 56

수정할 때 생각할 코드 구조

int sum = 0;

처음 m개를 sum에 더한다.
int max = sum;

for (int i = m; i < n; i++) {
    sum = sum + arr[i] - arr[i - m];
    max = Math.max(max, sum);
}
